Fonctionnalités techniques.

Le mode déconnecté permet l'utilisation de l'application sans connexion internet.

L'utilisation du service $resource d'AngularJS, la gestion de la mise à jour du cache, le HTML5 Web Storage et la compression (via d'autres modules de la communauté d'AngularJS) permet la persistance des données dans un navigateur internet.

Le transfert automatique des templates HTML vers un seul fichier JS (réalisé via un package de Node.js) évite les requêtes HTTP lors de l'utilisation de l'application.

La conversion automatique des medias en base 64 évite également les requêtes HTTP.

La détection automatique de la déconnexion à internet permet de basculer l'application en mode dégradé et d'en informer l'utilisateur.

Un extranet.

La méthode Digest de l'authentification HTTP permet de sécurisé tous les appels de l'application vers le serveur. Cela permet également d'éviter la ressaisie du mot de passe de l'utilisateur lors de ses prochaines utilisations.

L'accès aux vues de l'application est contrôlé par la promesse resolve de l'objet $routeProvider.

Certains profils utilisateurs peuvent réaliser des opérations CRUDS.

L'import et le parsage de fichiers spécifiques permet de faciliter la saisie des utilisateurs.

Visualisation et manipulation 3D avec WebGL.

Graphiques dynamiques avec D3.js (dot-plot, histogramme, arbre phylogénétique).

Un tour guidé de l'application.

Réaliser via un module de la communauté d'AngularJS, il permet de familiariser l'utilisateur à chaque vue de l'application à travers la réalisation d'exemples concrets.

Statistiques

Utilisation de Google tag manager via un module de la communauté d'AngularJS. Cela permet également de suivre finement chaque profil utilisateur.

Architecture applicative.

Anagène a été construite avec un générateur de Yeoman. Il permet notamment d'assurer l'ajout de contrôleur ou autres.

L'application est basée sur une architecture REST.

Le développement d'un module SSO pour serveur CAS (utilisation d'une machine à états finis) permet l'interconnexion avec un annuaire d'utilisateurs.

GruntJS fournie un serveur web local qui permet de contrôler instantanément le comportement de l'application sur plusieurs navigateurs internet.
Il permet également de facilité le déploiement de l'application au travers de différents services : l'injection automatique des composants Bower dans l'application, la minification des scripts JS, CSS et des images, l'ajout automatique des préfixes CSS pour la compatibilité des navigateurs web, la concaténation des fichiers, et l'utilisation de références CDN Google

La gestion de version des librairies JS est réalisée avec Bower. A noter que tout au long du projet la dernière version d'AngularJS et de ReactJS a été utilisée.

L'exécution de tests automatiques est réalisée avec Karma et Jasmine.

UX & UI.

Au-delà des fonctionnalités natives à AngularJS, l'utilisation des modules tel que x-editable et angular-strap (basés sur le framework CSS Bootsrap) améliore l'expérience de l'utilisateur. A noter ma contribution sur certains des modules utilisés dans l'application.

Test.

Les services, contrôleurs, directives et filtres d'AngularJS ont bénéficié de l'écriture de tests unitaires automatiques de non régression.

Au travers de ces tests, l'injection de différents jeux de données permet de tester à chaque déploiement l'application de A à Z.

L'écriture de tests fonctionnels contrôle plusieurs navigateurs internet et permet de vérifier le comportement de l'application.

AngularJS fournit toutes les fonctionnalités nécessaires pour tester correctement l'application. Par exemple, la gestion des requêtes asynchrones avec $httpBackend.

Performance.

En cours de développement, AngularJS Batarang permet de repérer les traitements les plus gourmands en mémoire et donc de les améliorer.

Le virtual scroll (implémenté via un module de la communauté) rend les listes longues performantes tout en augmentant la visibilité de l'utilisateur.

Pour les listes longues éditables, l'implémentation de ReactJS reste la solution la plus performante et maintenable.

L'utilisation de l'évènement $destroy évite les fuites de mémoire.

La transclusion augmente les performances, ainsi que le One-time binding.
